# 0008 期限超過の視覚演出

## Status

Accepted — `!!` 表示位置は [0009-overdue-mark-column.md](0009-overdue-mark-column.md) により Superseded

## Context

Deadline 列 (US-008) の追加により期限管理が可能になったが、期限超過の Todo を一覧上で即座に識別する手段がない。ユーザーが一目で超過を認識できる視覚的な演出が必要。

## Decision

期限超過の判定と演出を以下の方針で実装する。

**判定ロジック** (フロントエンドのみ):
- `deadline !== null && status !== "Done" && new Date(deadline) < new Date()` で超過と判定
- Done 状態の Todo は完了済みのため超過表示しない
- deadline が null (未設定) の Todo は超過扱いにしない

**演出**:
1. 行頭の `#` 列に行番号の代わりに赤い `!!` マークを表示
2. 行全体の背景色を薄い赤系 (`#fef2f2`) に変更

**代替案**:
- 行番号を残して別列にアイコン表示 → 列追加のコスト不要な現案を採用
- Deadline 列のテキスト色のみ変更 → 視認性が低いため不採用

## Consequences

- 期限超過の Todo が一覧上で即座に識別可能になる
- 判定はクライアントサイドのみで完結し、API 変更不要
- `new Date()` による判定のため、画面表示中にリアルタイムで切り替わることはない（リロードまたは操作時に再描画）
