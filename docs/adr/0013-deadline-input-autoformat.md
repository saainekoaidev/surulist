# 0013 Deadline入力の自動整形・バリデーション

## Status

Accepted

## Context

US-015 で `<input type="date/time">` から `<input type="text">` に変更した結果、ブラウザのカレンダーピッカーや入力制約がなくなり、不正な値 (存在しない日付、文字列など) を自由に入力できてしまう。また `20260507` のように数字だけ打って `yyyy/mm/dd` に自動変換してほしいという要望がある。

## Decision

commit タイミング (blur / Enter) で入力値をパース・バリデーションし、有効なら正規化、無効なら消去する。

### Date 入力の受理パターン

| 入力例 | 解釈 |
|--------|------|
| `20260507` | 8桁数字 → `2026/05/07` |
| `2026/05/07` | 区切り付き → そのまま |
| `2026-05-07` | ハイフン区切り → `2026/05/07` に変換 |
| `2026/5/7` | 1桁月日 → `2026/05/07` にゼロ埋め |
| `2026/02/29` | 存在しない日付 → 消去 |
| `abc` | 非数字 → 消去 |
| (空) | deadline = null |

### Time 入力の受理パターン

| 入力例 | 解釈 |
|--------|------|
| `1234` | 4桁数字 → `12:34` |
| `930` | 3桁数字 → `09:30` |
| `12:34` | コロン付き → そのまま |
| `9:30` | 1桁時 → `09:30` にゼロ埋め |
| `2500` | 範囲外 (時>23) → 消去 |
| `1260` | 範囲外 (分>59) → 消去 |
| (空) | 時刻なし (日付のみなら `T00:00`) |

### バリデーションタイミング

blur / Enter / 更新ボタン押下時。入力中 (onChange) では制約を掛けず自由入力を許容する。

## Consequences

- 数字のみの高速入力が可能になる (`20260507` → `2026/05/07`)
- 存在しない日付やフォーマット外の入力は自動的に消去され、不正データがサーバーに送られない
- カレンダーピッカーは使えないが、直接入力の利便性が向上する
