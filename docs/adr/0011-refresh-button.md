# 0011 Todo一覧のリフレッシュボタン

## Status

Accepted

## Context

期限超過の `!!` マーク表示と行背景の変更 (US-011, US-012) はページ描画時に評価されるが、画面を開いたまま時間が経過しても自動更新されない。また目的リストボックスの切り替え時にも再評価されない場合がある。

リアルタイム更新 (setInterval / WebSocket) は本アプリの規模に対してオーバーキルであるため、ユーザー起点の手動リフレッシュで対応する。

## Decision

Todo 一覧のセクションバー (`section-bar`) にリフレッシュボタンを追加する。

- **アイコン**: くるくる矢印のインライン SVG (テキストラベルなし)
- **配置**: `h2` + `badge` の右にフレックススペーサーを挿入し、ボタンをテーブル右端に揃える
- **動作**: 押下で `fetchTodos()` を呼び出し、Todo リスト全体を再取得。再取得後の描画で `isOverdue()` が再評価され、`!!` マーク・`row-overdue` 背景が最新の時刻で判定される
- **Props**: `TodoTable` に `onRefresh?: () => void` を追加。`App.tsx` から `fetchTodos` を渡す

### 代替案

- `setInterval` で定期的にリフレッシュ → 不要な通信が発生。localhost 用途でも無駄
- `isOverdue` のみクライアント側で再計算 (fetch なし) → サーバー側の変更 (他ブラウザ/タブでの更新) を反映できない

## Consequences

- ユーザーが任意のタイミングで期限超過表示を更新できる
- ボタン 1 つの追加のみで、既存のデータフロー (`fetchTodos` → state → 再描画) を再利用できる
- リアルタイム性は犠牲になるが、本アプリの用途 (個人用 localhost) では許容範囲
