# 0009 期限超過マークの専用列化

## Status

Accepted

Supersedes: [0008-overdue-highlight.md](0008-overdue-highlight.md) の `!!` 表示位置に関する部分

## Context

US-011 で `#` 列の行番号を `!!` マークに置き換える方式を採用したが、行番号が消えると一覧性が落ちるというフィードバックがあった。また `!!` が Todo テキストと視覚的に離れており、どのタスクが超過しているか直感的に分かりにくい。

## Decision

Status 列と Todo 列の間に幅 28px の専用列 `(超過)` を新設し、期限超過時に `!!` マークを表示する。`#` 列は常に行番号を表示する。

- ヘッダには列名を表示しない (空 `<th>`)
- colspan 参照は 7 → 9 に更新 (超過列 + Date/Time 分割による 2 列増)
- TodoNewRow にも空 `<td>` を追加してカラム数を合わせる
- 行背景の `row-overdue` クラスは US-011 から変更なし

## Consequences

- 行番号が常に表示され、一覧性が維持される
- `!!` マークが Todo テキストの直前に表示され、超過の認知が向上する
- テーブル列数が 7 → 9 に増加するが、超過列は 28px と狭く、Date/Time 分割で旧 Deadline 170px を 130px+90px=220px に再配分するため、レイアウトへの影響は許容範囲
