## 2. 画面遷移

本アプリは SPA 単一画面構成 (US-001)。ルーティングによる画面遷移は存在しない。
画面内の状態遷移 (ビュー切替) のみで全操作を完結させる。

```mermaid
stateDiagram-v2
    [*] --> S01_通常表示

    state S01_メイン一覧 {
        S01_通常表示 --> S01_新規追加行 : [新規追加] クリック
        S01_新規追加行 --> S01_通常表示 : [登録] or [キャンセル]
        S01_通常表示 --> S01_編集モード : Todo テキストをクリック
        S01_編集モード --> S01_通常表示 : [更新] or [キャンセル]
        S01_通常表示 --> D01_目的編集 : [編集] クリック
        D01_目的編集 --> S01_通常表示 : [閉じる]
    }
```

## 3. 各画面の仕様

### S01 メイン一覧

- **目的**: Todo の一覧表示・CRUD 操作と、目的 (Category) の切替を行う
- **対応 US**: US-001, US-002, US-004, US-005, US-006, US-008, US-009, US-010, US-011, US-012, US-013, US-014, US-015

#### 3-1. ヘッダ

| 要素 | 種別 | 仕様 |
|------|------|------|
| アプリタイトル | テキスト | 「するリスト」固定表示。青背景・白文字 |

#### 3-2. ツールバー

| 要素 | 種別 | 仕様 |
|------|------|------|
| Objective リストボックス | `<select>` | 登録済み目的を一覧表示。選択切替で紐づく Todo を再取得・再描画 (US-002)。全件表示オプション有効時は末尾に「(全て)」選択肢を追加 (US-007) |
| 編集ボタン | `<button>` | 目的編集ダイアログ (D01) を開く (US-003) |
| 新規追加ボタン | `<button>` | Todo 一覧の先頭にインライン入力行を挿入 (US-004)。全件表示モード時は disabled |

#### 3-3. Todo 一覧テーブル

**件数表示**: テーブル上部に「N件」バッジを表示 (US-005)。

**リフレッシュボタン** (US-014): セクションバー右端にくるくる矢印アイコンのボタンを配置。押下で Todo リストを再取得し、Deadline 比較を再実行して `!!` マーク・背景色を更新する。ラベルは表示せずアイコンのみ。

| 列 | データ | 幅目安 | 備考 |
|----|--------|--------|------|
| # | 連番 (表示用) | 36px | 自動採番 (1-origin) (US-015: 44px→36px) |
| Status | 進捗リストボックス | 140px | 常時変更可能。値変更で即 PUT 発行 (US-006) (US-015: 160px→140px) |
| (超過) | 期限超過マーク | 28px | 超過時に赤い `!!` を表示。通常時は空欄 (US-012) |
| Todo | テキスト | 残り幅 | 全角 40 文字上限 (US-004)。クリックで編集モードへ (US-005)。フォント 13px (US-015: 14px→13px) |
| Date | 日付入力 | 120px | `<input type="text">`。`yyyy/mm/dd` 形式で表示。オプション (null 許容)。フォント 12px (US-015: 13px→12px, 130px→120px) |
| Time | 時刻入力 | 80px | `<input type="text">`。`hh:mm` 形式。Date と合わせて deadline を構成。フォント 12px (US-015: 13px→12px, 90px→80px) |
| RegistDate | 登録日時 | 95px | `YYYY/M/D H:mm` 形式。読み取り専用。フォント 11px (US-015: 110px→95px) |
| UpdateDate | 更新日時 | 95px | 同上。未更新時は空欄。フォント 11px (US-015: 110px→95px) |
| (操作) | ボタン群 | 110px | 状態に応じて表示切替 (下記) (US-015: 130px→110px) |

#### 3-4. 行の状態と操作ボタン

| 状態 | 表示されるボタン | トリガー → アクション |
|------|------------------|----------------------|
| 通常表示 | なし | 行の Todo テキストをクリック → 編集モードへ |
| 新規追加行 | [登録] | [登録] クリック → `POST /api/todos` → 行を除去して一覧再描画 |
| 編集モード | [更新] [削除] | [更新] → `PUT /api/todos/:id` / [削除] → 確認後 `DELETE /api/todos/:id` |

> 更新・削除ボタンは普段は非表示。編集モードに入った場合のみ表示。

#### 3-5. ステータスチップ

| DB 値 | 表示ラベル | チップ色 |
|-------|------------|----------|
| `Not Started` | Not Started (未着手) | グレー |
| `In Progress` | In Progress (進行中) | ブルー |
| `Pending` | Pending (保留) | イエロー |
| `Done` | Done (完了) | グリーン |

Status は完了状態でも Todo の編集・削除を制限しない (US-006)。

#### 3-6. 全件表示モード (US-007)

「(全て)」選択時は、全目的の Todo をカテゴリごとにグループ分けして表示する。

| 要素 | 仕様 |
|------|------|
| グループ見出し行 | 各カテゴリの先頭に `colspan=9` の見出し行を挿入。カテゴリ名を表示 |
| 行番号 (#) | カテゴリごとに 1 から振り直し |
| 新規追加ボタン | disabled（追加先カテゴリが不明のため） |
| 編集・削除・ステータス変更 | 通常通り操作可能 |

#### 3-7. 期限超過演出 (US-011, US-012)

Deadline が現在時刻を超過している行に視覚的な警告を表示する。

| 要素 | 条件 | 表示 |
|------|------|------|
| (超過) 列 | 超過時 | Status と Todo の間の専用列に赤い `!!` マークを表示 (US-012) |
| 行背景 | 超過時 | 薄い赤系背景 (`--color-overdue-bg: #fef2f2`) |
| Done 行 | 超過しない | Status が Done の場合は超過表示しない |
| # 列 | 常に行番号 | 超過有無にかかわらず連番を表示 (US-012 で変更) |

#### 3-8. Deadline 入力の遅延コミット (US-013)

Deadline の日付・時刻入力は即時反映ではなく、ユーザー操作完了時に確定する。

| 要素 | 仕様 |
|------|------|
| 入力方式 | `<input type="text">` × 2 列 (Date: `yyyy/mm/dd` 形式、Time: `hh:mm` 形式) |
| 確定タイミング | Enter 押下 / blur (フォーカス外れ) / 更新ボタン押下 |
| ローカル状態 | 入力中は React ローカル state で保持し、確定時のみ PUT 発行 |
| 値のクリア | 日付欄をクリアすると deadline = null で PUT 発行 (時刻のみでは deadline にならない) |

#### 3-9. テーブル視覚コンパクト化 (US-015)

| 項目 | 仕様 |
|------|------|
| セル padding | 上下 10px・左右 6px (従来: 左右 12px) |
| Date/Time 入力 | `<input type="text">` で `yyyy/mm/dd` / `hh:mm` 形式を直接表示 (ブラウザのロケール依存を排除) |

#### 3-10. バリデーション / エラー

| 対象 | ルール | エラー表示 |
|------|--------|------------|
| Todo テキスト | 必須 / 最大 40 文字 (全角換算) | 入力欄下にインラインメッセージ |
| Todo 削除 | 常に確認ダイアログ | 「削除してよろしいですか?」 |

### D01 目的編集ダイアログ

- **目的**: 目的 (Category) の追加・修正・削除を行う
- **対応 US**: US-003
- **表示方法**: S01 上にオーバーレイ付きモーダルとして重畳表示

#### 要素一覧

| 要素 | 種別 | 仕様 |
|------|------|------|
| ダイアログタイトル | テキスト | 「目的の管理」固定表示 |
| 目的リスト | テーブル | 登録済み目的を一覧表示。各行に目的名・件数・削除ボタン |
| 目的名 | テキスト / 入力欄 | クリックで編集可能。確定時に `PUT /api/categories/:id` |
| 件数 | テキスト | 紐づく Todo 件数を表示 (読み取り専用) |
| 削除 (✕) ボタン | `<button>` | 確認ダイアログ後、`DELETE /api/categories/:id` (Cascade 削除) |
| ＋ 追加ボタン | `<button>` | リスト末尾に入力行を追加。入力確定で `POST /api/categories` |
| 閉じるボタン | `<button>` | ダイアログを閉じる。S01 の Objective リストボックスを再取得して同期 |

#### オプションセクション (US-007)

目的リストと閉じるボタンの間に配置する。

| 要素 | 種別 | 仕様 |
|------|------|------|
| 全件表示を許可する | `<input type="checkbox">` | チェック ON で Objective リストボックスに「(全て)」選択肢を追加。設定は localStorage に保存 (DB 不使用、キー: `surulist:showAllEnabled`) |

#### バリデーション / エラー

| 対象 | ルール | エラー表示 |
|------|--------|------------|
| 目的名 | 必須 / 重複不可 | ダイアログ内にインラインメッセージ |
| 目的削除 | 紐づく Todo がある場合は確認ダイアログ | 「N件の Todo も削除されます。よろしいですか?」 |

## 4. 共通 UI

### 4-1. レイアウト

- 最大幅 1100px、中央寄せ (US-010: 960px から拡大)
- 背景: 薄いブルー (`#f0f6ff`)

### 4-2. デザインシステム

一次正: `frontend/src/styles/app.css`
HTML モックアップの Design Token (`docs/ui-images/main-list.html` の `:root` 変数) を参照元とする。

| トークン | 値 | 用途 |
|----------|----|------|
| `--color-primary` | `#3b82f6` | ボタン・アクセントカラー |
| `--color-bg` | `#f0f6ff` | ページ背景 |
| `--color-surface` | `#ffffff` | カード・テーブル背景 |
| `--color-border` | `#bfdbfe` | ボーダー |
| `--color-header-bg` | `#3b82f6` | ヘッダ背景 |
| `--color-editing-bg` | `#fef9c3` | 編集モード行の背景 |

### 4-3. フォントサイズ階層 (US-009)

| 要素 | サイズ | 根拠 |
|------|--------|------|
| Todo テキスト | 13px | 主要情報。最も大きく表示 (US-015: 14px→13px) |
| Date / Time | 12px | 補助情報。Todo より一段小さく (US-015: 13px→12px) |
| RegistDate / UpdateDate | 11px | メタ情報。最も控えめに |
| テーブルヘッダ (一般) | 12px | 列見出し |
| テーブルヘッダ (Date/Time) | 10px | Date/Time 列に合わせて縮小 (US-015: 11px→10px) |
| テーブルヘッダ (RegistDate/UpdateDate) | 10px | 日付列に合わせて縮小 |

### 4-4. 共通コンポーネント

| コンポーネント | 用途 |
|----------------|------|
| ヘッダ | アプリタイトル表示 (フッタなし) |
| 確認ダイアログ | 削除操作時の確認用。ブラウザ標準 `confirm()` or カスタムモーダル |
| バッジ | 件数表示 (`N件`) |
| ステータスチップ | 進捗状態の色付きラベル |
