# 要件定義

> 各 User Story (US) は `US-NNN	<本文>` (タブ区切り) の 1 行で記述する。新しい US は冒頭に追記して逆時系列順に並べる。
> 1 US 1 Issue 1 PR を原則とする (詳細は `CLAUDE.md` の per-US merge ceremony を参照)。

US-011	Deadlineが現在時刻を超過している場合, 行頭の#列に赤い「!!」マークを表示し, 行全体を薄い赤系背景に変更する。ただしDone状態のTodoは超過表示しない。
US-010	Todoの入力枠が小さいため, アプリ全体の最大幅を960pxから1100pxに拡大して, Todo列の表示・入力領域を広げる。
US-009	Todoテーブルのフォントサイズに視覚的階層を設ける。Todo(14px) > Deadline(13px) > RegistDate/UpdateDate(11px)。ヘッダも同様の階層を適用する。
US-008	Todoテーブルに「Deadline」列をTodo列の直後に追加する。ネイティブの日時入力(input type="datetime-local")を使用し, クリックでカレンダーが表示される。期限はオプション(null許容)とする。値変更で即PUT発行。
US-007	目的選択のリストボックスに「(全て)」の選択肢を追加し, 全目的の Todo をまとめて表示する機能を追加する。「(全て)」が表示されるかどうかは, 目的の管理ダイアログ内のオプション「全件表示を許可する」チェックボックスで制御する (UI設定, localStorage保存)。「(全て)」選択時は, Todo を目的ごとにグループ分けして表示し, 新規追加ボタンは無効化する。
US-001	俗に言うTodoリストを作りたいが, 画面は一覧画面の1画面構成で良い。画面遷移は極力不要なつくりにしたい。
US-002	一覧画面の左上に目的を選択するためのリストボックスを用意し, 選択切り替えで目的にぶら下がる一覧(Todoリスト)を表示させたい。
US-003	目的を選択するためのリストボックスの右に[編集]ボタン。押下すると, 目的を登録･編集するためのダイアログを表示。目的の追加･修正･削除を行えるようにする。修正してもぶら下がるTodoリストの内容は変わらないが, 削除すると紐づくTodoリストも削除されるようにする。目的の右には登録されているTodoリストの件数を表示。
US-004	一覧画面にTodoリストの一覧を表示する。[新規追加]ボタン押下で行追加され, Todoの入力を行の上で可能にする。全角40文字位をリミットにすることで良い。行の右端に[登録]ボタンを用意し, 押下によってInsertが走るようにする。
US-005	登録されたTodoを一覧に表示する。件数も表示してほしい。行の構成は #･進捗リストボックス・Todo・登録日時・更新日時 の5つ＋後述するボタン群。Todoを選択すると入力(編集)が可能となり, Todoを修正できるようになる。行の右端に[更新]ボタンと[削除]ボタン。更新押下でUpdate、削除押下でDelete.
US-006	進捗リストボックスの中身(ステータス)はAIが決めてください。英語(日本語)で出してくれると格好良いと思います。なお, 完了に相当する進捗状況でも, Todoの編集や削除は可能とする。
